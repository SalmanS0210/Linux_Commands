1. The Linux Filesystem Hierarchy
Unlike Windows, which uses separate drive letters (C:, D:), Linux organizes everything into a single inverted tree structure starting from the Root directory (/). Every single file, folder, and device is a branch on this tree.
Key Directories You Must Know:

/etc: The central command post. This contains almost all system and application configuration files (e.g., /etc/passwd, /etc/ssh/sshd_config)
/home: The personal quarters. Every standard user gets a subdirectory here (e.g., /home/john) to store their personal files.
/var: Variable data. This is where files that constantly change live—most importantly, system logs (/var/log).
/bin & /sbin: Where the power tools live. These contain essential command binaries (like ls, cd, grep) needed to boot and run the system.

Core Navigation Commands:Bash
pwd              # "Print Working Directory" - shows exactly where you are standing
ls -la           # List all files, including hidden ones (filenames starting with a dot)
cd /etc          # Change directory to /etc (Absolute path)
cd ../           # Move up one level (Relative path)
cd -             # Toggle back to the previous directory you were in


2. Linux Permissions (The Security Gate)
Linux is built from the ground up to be a multi-user system, meaning security boundaries are strictly enforced. Every file and directory has an owner and a specific set of permissions.
If you run ls -l, you will see a string that looks like this: -rwxr-xr--

Breaking Down the String:
The First Character: Tells you the file type (- means regular file, d means directory).
The Three Triplets: * User (Owner): What the person who owns the file can do.
         Group: What members of the file's assigned group can do.
         Others: What everyone else on the system can do.

The Two Ways to Change Permissions (chmod):
Symbolic Mode (Beginner Friendly): Uses letters to add (+) or remove (-) permissions.
Bash
chmod +x script.sh      # Give everyone execute permissions
chmod g-w report.txt    # Remove write permission from the group

Absolute (Numeric) Mode (Pro Level): 

Uses a 3-digit octal number representing User, Group, and Others.
read (r) = 4
write (w) = 2
execute (x) = 1

Add them together to get your number. For example: rwx ($4+2+1=7$) and r-x ($4+0+1=5$).
Bash
chmod 755 script.sh     # Owner: rwx (7), Group: r-x (5), Others: r-x (5)
chmod 644 config.txt    # Owner: rw- (6), Group: r-- (4), Others: r-- (4)

To change who owns the file, use chown:
Bash
sudo chown john:developers file.txt   # Changes owner to 'john' and group to 'developers'

3. Users & Groups
Every process that runs on Linux runs under the context of a user.

Critical User Types:
Root User (UID 0): The absolute administrator. Root can bypass all permission checks and delete the entire system with a single command.
System Users (UID 1-999): Non-human accounts used by services (like nginx or mysql) so they don’t have to run with dangerous root privileges.
Regular Users (UID 1000+): Real people (like you!).
Managing Accounts:
Bash
sudo useradd -m -s /bin/bash alice   # Create user 'alice' with a home directory and bash shell
sudo passwd alice                    # Set or change Alice's password
sudo groupadd DevOps                 # Create a new group called 'DevOps'
sudo usermod -aG DevOps alice       # Append (-a) Alice to the 'DevOps' group (-G)

The Power of sudo
Because logging in as root is a massive security risk, standard administrators use sudo (Superuser Do). This allows a regular user to execute a single command with root privileges, which is logged for accountability.To give a user sudo rights on Ubuntu, you simply add them to the system's sudo group:
Bash
sudo usermod -aG sudo alice
